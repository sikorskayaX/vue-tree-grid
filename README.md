## Класс TreeStore

### Интерфейс элемента

```ts
type TreeItemId = string | number

interface TreeItem {
  id: TreeItemId
  parent: TreeItemId | null
  label?: string
  [key: string]: unknown  // произвольные поля
}
```

### Методы

| Метод | Описание | Сложность |
|---|---|---|
| `getAll()` | Возвращает все элементы | O(n) |
| `getItem(id)` | Возвращает элемент по id | O(1) |
| `getChildren(id)` | Возвращает прямых потомков | O(k) — k детей |
| `getAllChildren(id)` | Возвращает всех потомков (рекурсивно) | O(N) — N в поддереве |
| `getAllParents(id)` | Возвращает цепочку от элемента до корня | O(h) — высота дерева |
| `addItem(item)` | Добавляет новый элемент | O(1) |
| `removeItem(id)` | Удаляет элемент и всех потомков | O(N) — N в поддереве |
| `updateItem(item)` | Обновляет элемент | O(1) |

### Внутреннее устройство

TreeStore использует три структуры данных для обеспечения производительности:

- **`itemsMap`** (`Map<TreeItemId, TreeItem>`) — O(1) доступ к любому элементу по id
- **`childrenMap`** (`Map<TreeItemId, TreeItemId[]>`) — списки прямых потомков для каждого узла
- **`rootIds`** (`TreeItemId[]`) — элементы без родителя (корни дерева)

Конструктор строит все три структуры за один проход по массиву (O(n)).

## Vue-компонент TreeTable

Компонент использует AG Grid Enterprise с режимом `treeData: true` для отображения иерархических данных.

### Колонки

- **№ п/п** — порядковый номер строки в таблице
- **Элемент** — древовидная колонка (с expand/collapse), отображает `label`
- **Категория** — определяет тип строки: «Группа» (есть дети) или «Элемент» (лист)
- **ID** — идентификатор элемента
- **Родитель** — идентификатор родителя
- **Действия** — кнопки редактирования и удаления элемента

### Взаимодействие

- **Добавление**: кнопка «+ Добавить элемент» открывает форму с полями ID, Parent, Label
- **Удаление**: кнопка ✕ в колонке «Действия» удаляет элемент и всех его потомков
- **Редактирование**: кнопка ✎ открывает модальное окно для изменения Parent и Label (ID изменять нельзя)

### Пропсы

```ts
interface Props {
  items: TreeItem[]
}
```

## Тестирование

### TreeStore (35 тестов)

- `getAll()` — длина, порядок элементов
- `getItem()` — числовой id, строковый id, несуществующий id
- `getChildren()` — корень, промежуточный узел, глубокий узел, лист, несуществующий id
- `getAllChildren()` — корень, промежуточный узел, лист, несуществующий id
- `getAllParents()` — цепочка от листа до корня, промежуточный узел, корень, несуществующий id
- `addItem()` — новый корень, новый потомок, без label
- `removeItem()` — лист, узел с потомками, корень со всем поддеревом, несуществующий id
- `updateItem()` — смена label, смена родителя, перемещение в корень, несуществующий id

### TreeTable (7 тестов)

- Монтирование без ошибок
- Наличие CSS-класса `ag-theme-quartz`
- Рендер AG Grid (`.ag-root`)
- Заголовки колонок (включая «Действия»)
- Количество строк
- Отображение категорий «Группа» / «Элемент»
- Переключение формы добавления
